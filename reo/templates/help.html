REopt API

The REopt API provides access to a web version of a techno-economic decision support model used to optimize energy systems for buildings, campuses, communities, and microgrids. REopt recommends an optimal mix of renewable energy, conventional generation, and energy storage technologies to meet cost savings and energy performance goals.

The REopt API uses utility rates from the Utility Rate Database and coordinates with PV Watts for solar resource information. It is capable of accepting custom rate profiles, but is also equipped with Department of Energy Commercial Reference Buildings.

Input Format

The REopt API accepts a JSON post in a nested directory format. At each level in the directory, capitalized directory names are associated with sets of lower case attribute names and values (as well as possible subdirectories). Only required attributes names and values need to be initially provided as missing attribute values will be filled in with default values as described in the tables below.

Note: Optimizations include PV and Storage capabilities by default. To disable either, set its 'min_kw' attribute to 0.

Below is an example of a a simple valid post that disbales storage in the optimization: 

{"Scenario":{
		"Site":{  
			"latitude": 39.5,
			"longitude": -105.0,  
			"Financial":{},  
			"LoadProfile":{  
				"annual_kwh": 20000,  
				"doe_reference_name": 'RetailStore'   
			},  
			"ElectricTariff":{
				"blended_monthly_rates_us_dollars_per_kwh": [0.12,0.12,0.12,0.12,0.12,0.12,0.12,0.12,0.12,0.12,0.12,0.12]  
				"monthly_demand_charges_us_dollars_per_kw": [8,8,8,8,8,8,8,8,8,8,8,8]
			},  
			"PV":{},  
			"Storage":{  
				'min_kw': 0  
			}  
			}  
	    }  
}
